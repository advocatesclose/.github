# ADVOCATES CLOSE CORPORATION

If you save a document to Word and then save it to PDF, you did not do two different things exaclty one time. You did exactly one thing two different times. And yet, in practice, no one would ever see it that way. In practice, you did two things--that is why you have two documents. And therein lies the problem.

The problem is that, right now, there is a one-to-one correspondence between the work that you do and the result that you get. You press "save" one time so that you may save one document. You press "run" one time so that you may run one program. And so on. The whole point of this app is to break that correspondence.

There is no reason why inputs should be related to outputs in this way.
